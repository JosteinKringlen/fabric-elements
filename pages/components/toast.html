Toast

Props

Type of toast: type — default success (success, warning, error, info)

Text content: text — default undefined

Auto removal: duration — default 2400 (only used for programmatic toast duration)

Close handler: onClose — default undefined (not needed, but will perform alongside close)

Toast message: text — default undefined (only used when updating toast)

Programmatic toasting

import { toast, removeToast, updateToast } from '@fabric-ds/elements/toast';

Easiest way to emit a successful toast with auto removal

toast('message')

Setting duration to 0 will disable auto-removal. The canClose property tells the toast that it can be closed, removing this and setting duration to 0 creates a toast that never goes away.

toast('message', { type: 'warning', duration: 0, canClose: true })

The toast() function returns the toast, meaning you can assign this to a variable and update / remove the toast later.

const toast = toast('message', { type: 'warning', duration: 0 }) updateToast(toast.id, { text: 'Updated the toast message and type', type: 'error' }); removeToast(toast.id):
